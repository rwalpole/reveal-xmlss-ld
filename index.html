Semiotic triangle

Formal knowledge representation

Ambiguity makes natural language difficult
Machines need a formal knowledge representation
Unambiguous concepts, relationships and entities
Semantics relates to meaning in language and logic
Semantics depends on context and pragmatics

Context and Pragmatics

If I say.. "I could use some coffee"
You need to know the context
Are we in a supermarket.. or a classroom?
You use pragmatics to analyse the context
And deduce that I would like to drink some coffee ☕

The triple

A simple sentence has a subject, predicate and object.
For example..
St Edmund Hall is part of Oxford Univeristy
Oxford University is in Oxford
Terry Jones studied at St Edmund Hall

The graph

subject → predicate → object
node → edge → node = graph
The edge is directed so we have a directed graph
The edge also has a label so we have a
directed edge-labeled graph

Entities and relations

St Edmunds Hall, Oxford University, Terry Jones
Nodes are entities
is in, is part of, studied at
Edges are relations

Literals and classes

Terry Jones → born in → 1942
Oxford → total population → 151,584
These are not entities but literals
Terry Jones → is a → Person
Oxford → is a → City
These are not entities but classes

What is a knowledge graph?

“A knowledge graph (i) mainly describes real world entities and their interrelations, organized in a graph, (ii) defines possible classes and relations of entities in a schema, (iii) allows for potentially interrelating arbitrary entities with each other and (iv) covers various topical domains.”

Heiko Paulheim (2017)

Uniform Resource Locator

Addresses for web documents
URL identifies what exists on the web (document)
URI identifies on the web what exists (anything)
So how do we talk about apples on the Web?

URI for an apple?

http://example.org/apple

What about this apple?

Language independent URIs

Apple the fruit on Wikidata:
https://www.wikidata.org/entity/Q89
Apple the company on Wikidata:
https://www.wikidata.org/entity/Q312

Designator and designatum

The apple itself cannot be on the Web
But we can describe the apple on the Web
The URI refers to the apple
The URI is a designator
The apple is the designatum

Content Negotiation

Accessing the URI might return some representation of the apple
HTTP content negotiation allows you to chose the representation you want

						curl -L -H "Accept: text/html" http://wikidata.org/entity/Q89
						curl -L -H "Accept: application/json" http://wikidata.org/entity/Q89
						curl -L -H "Accept: text/turtle" http://wikidata.org/entity/Q89

Representing graphs and triples with RDF

subject → predicate → object

St Edmund Hall → is part of → Oxford University
If we want to express this in RDF we need to use URIs
In RDF the predicate is known as a Property
Subject(URI) → Property (URI) → Object/Value (URI/Literal)
<http://dbpedia.org/resource/University_of_Oxford>
<http://dbpedia.org/resource/location>
<http://dbpedia.org/resource/Oxford>

RDF Schema

A data modelling vocabulary for RDF
Allows you to define classes, properties and instances
Instances are indivual members of a class

Defining classes and instances


							@prefix dbo: <http://dbpedia.org/ontology/> .
							@prefix rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#> .

							dbo:University rdf:type rdf:Class .
							dbo:University rdfs:subClassOf dbo:EducationalInstitution .
							dbr:University_of_Oxford rdf:type dbo:University .

Defining properties


							@prefix dbo: <http://dbpedia.org/ontology/> .
							@prefix rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#> .

							dbo:district rdf:type rdf:Property ;
							             rdfs:domain dbo:Place ;
													 rdfs:label "district"@en ;
													 rdfs:range dbo:PopulatedPlace
													 rdfs:subPropertyOf dbo:isPartOf .

Other things to note

  • Classes start with uppercase
  • Properties start with lowercase
  • rdf:type can be abbreviate as 'a'

							@prefix dbo: <http://dbpedia.org/ontology/> .
							@prefix rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#> .

							dbo:University a rdf:Class .
							dbo:University rdfs:subClassOf dbo:EducationalInstitution .
							dbr:University_of_Oxford a dbo:University .

Linked Data Principles

  1. Use URIs as names for things.
  2. Use HTTP URIs so that people can look up those names.
  3. When someone looks up a URI, provide useful information, using the standards (RDF, SPARQL).
  4. Include links to other URIs, so that they can discover more things.
http://www.w3.org/DesignIssues/LinkedData.html
Tim Tim Berners-Lee, 27th July 2006

5 star linked open data

  1. ⭐ Available on the web (whatever format) but with an open licence, to be Open Data
  2. ⭐ Available as machine-readable structured data (e.g. excel instead of image scan of a table)
  3. ⭐ as (2) plus non-proprietary format (e.g. CSV instead of excel)
  4. ⭐ All the above plus, Use open standards from W3C (RDF and SPARQL) to identify things, so that people can point at your stuff
  5. ⭐ All the above, plus: link your data to other people’s data to provide context
